# SOP: 组合全面体检 (Portfolio Checkup)

当用户并未指定具体股票，而是要求你对其**整体投资组合**进行综合评估、风险分析或健康度检查时，你**必须严格执行以下串行流程**。

## 阶段 1：数据采集 (Data Gathering)

请按顺序调用以下工具，一次性收集组合的多维度数据：

1. **组合全景数据（配置/集中度/Beta/基准/归因/回撤/相关性）**：
   ```bash
   ./ibkr portfolio all --json
   ```

2. **持仓技术面批量扫描**：
   从上一步的数据中提取所有股票持仓代码（仅 STK 类型），然后一次性分析：
   ```bash
   ./ibkr analyze SYMBOL1 SYMBOL2 SYMBOL3 ... --json
   ```

3. **财报事件风险排查**：
   ```bash
   ./ibkr earnings portfolio --json
   ```

4. **板块轮动对比（可选，如持仓分布跨多个板块时推荐）**：
   ```bash
   ./ibkr sectors --json
   ```

## 阶段 2：交叉分析 (Cross Analysis)

在你的思维链中，核对以下关键维度并形成判断：

- **集中度 vs 相关性**：如果 HHI 偏高，且高权重持仓之间还高度相关（>0.7），风险叠加严重
- **技术面强弱分布**：从技术评分中找出最强和最弱的持仓，评估是否需要"汰弱留强"
- **财报地雷**：近 7 天内有财报的持仓，提醒用户注意波动风险
- **Alpha 来源**：从盈亏归因中找出拖累和贡献最大的持仓
- **Beta 暴露**：组合 Beta 是否符合用户的风险偏好（>1.2 偏激进，<0.8 偏保守）

## 阶段 3：报告输出 (Report Delivery)

请以**专业投资顾问**的口吻，按以下格式结构化输出：

```markdown
# 🏥 组合健康度体检报告

## 1. 📊 组合概览
- 净值 / 现金占比 / 持仓数量
- 组合 Beta（对市场的敏感度判断）

## 2. ⚖️ 配置分析
- 按资产类型分布（股票 vs 期权 vs 现金）
- 按行业板块分布（是否过度集中某一板块）

## 3. ⚠️ 风险维度
- HHI 集中度评估
- 高相关性持仓对预警
- 近期财报事件风险

## 4. 📈 绩效评估
- vs SPY 的 Alpha
- 盈亏归因 Top 3（贡献最大 & 拖累最大）
- 组合最大回撤

## 5. 💪 技术面强弱分布
- 最强持仓 Top 3（评分、信号）
- 最弱持仓 Top 3（评分、信号）

## 6. 💡 改进建议
- 基于以上数据的具体操作建议（如：降低某只集中度、关注财报前减仓等）

⚠️ 以上分析仅供参考，不构成投资建议
```
